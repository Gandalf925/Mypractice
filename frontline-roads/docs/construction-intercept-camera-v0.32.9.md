# FRONTLINE ROADS v0.32.9 建設圏適正化・敵部隊迎撃・カメラ移動

## 1. 建設圏の再設計

文明レベルごとの2倍成長を廃止し、レベル0〜4の明示的な上限付きテーブルへ置き換えた。

| 文明Lv. | 主要拠点 | 簡易拠点 |
|---:|---:|---:|
| 0 | 85m | 50m |
| 1 | 120m | 75m |
| 2 | 160m | 105m |
| 3 | 205m | 140m |
| 4 | 255m | 180m |

レベル上限を超える値も最終段階へ固定する。現在地は85m、出撃中の遠征部隊は120mのままであり、文明後半でも初期保持半径1,250m全体を拠点から建設できない。前線への移動、簡易拠点、遠征部隊の役割を維持する。

旧 `constructionRangeMultiplier` とアンカー内の未使用 `baseRange` / `rangeMultiplier` は削除した。名称、拠点司令部の説明、設置失敗理由、描画範囲は同じ定義を参照する。

## 2. 敵部隊への迎撃派兵

移動中で出発待機を終えた敵マーカーを選択すると「この敵部隊へ派兵」が表示される。派兵後は `INTERCEPT` 任務として選択敵IDを保持する。

- 敵の現在経路における次ノードを追跡地点にする。
- 敵が次区間へ移ると味方経路を再計算する。
- 近接時は指定敵を他の敵より優先して攻撃する。
- 対象が撃破・消滅した場合は元の拠点へ自動帰還する。
- 回収任務および敵拠点攻撃とは状態を分離し、連携出撃は敵拠点攻撃だけに限定する。

## 3. カメラ即時移動

通常戦闘HUDへ次の独立操作を追加した。

- `＋` / `−`: 現在の画面中央を基準に拡大・縮小。
- `拠`: 拠点司令部で現在選択されている拠点へ移動。未選択時は利用可能な先頭拠点。
- `現`: GPSから反映されたプレイヤー現在地へ移動。

初期拠点選択画面の地図操作とは別要素にし、縦画面・低い画面・横画面で施設ツール列と重ならない配置を定義した。

## 4. 互換性

セーブキーとセーブスキーマは変更していない。既存の攻撃任務は `ATTACK` のまま読み込まれ、`targetEnemyId` がない既存部隊へ迎撃状態を付与しない。
