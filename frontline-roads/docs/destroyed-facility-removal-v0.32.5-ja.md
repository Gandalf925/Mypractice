# FRONTLINE ROADS v0.32.5 破壊施設の即時撤去

## 仕様変更

- 戦闘設備、支援設備、木製観測所、防壁、門はHPが0になった時点で状態配列から即時削除されます。
- 集落内の生産施設もHPが0になった時点で即時削除され、対応する生産キューも同時に破棄されます。
- 破壊地点のノードまたは道路はその場で建設可能になります。
- 門が破壊された場合、道路封鎖は即時解除され、敵は再経路計算後に通行します。
- 修理できるのはHPが1以上残っている損傷施設だけです。HPが0になった施設を修理する経路はありません。

## 既存セーブの整理

ロード時に次の処理を一度だけ行います。

1. `ruined`またはHP 0以下の戦闘設備を削除。
2. `ruined`、`demolished`、またはHP 0以下の集落施設を削除。
3. 削除された集落施設に属する生産キューを削除。
4. 同じノードまたは道路上に重なった有効施設を1基へ整理し、保存データ上で最後に記録された有効施設を残す。
5. 削除された設備を狙っていた敵の対象を解除し、経路再計算を要求。

セーブキーとスキーマは変更していません。

## 削除した不要処理

- 残骸を修理可能な状態として保持する処理。
- 破壊済み門・施設の専用描画。
- `FIX`、`OPEN`表示。
- 残骸優先選択、重複残骸の選択切り替え。
- 残骸の修理・撤去分岐。
- 修理後の再稼働待機処理。
- 集落施設の`ruined`・`demolished`常駐状態。
- 残骸数を基準にしたオフライン損失集計。
- 重複していた防衛対象経路無効化処理。

旧セーブを安全に整理するため、`ruined`と`demolished`の読み取りはロード時の正規化と旧形式移行にだけ残しています。
